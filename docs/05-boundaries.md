# 05. 公開/非公開の線引き宣言

このリポジトリは、**何を公開し、何を公開しないか**を明示する。透明性のためであり、同時に「どこまでが無料で、どこからが支援か」の線引きでもある。

## 3 層構造

```
┌─ 公開層（本リポジトリ｜OSS）───────────────────────┐
│  ソルバー実装・BP-10 テンプレ・メソッド選定決定木・思想        │
└──────────┬──────────────────┬──────────────┘
           ↓ 参照                    ↓ 参照
┌─ 診断ツール層 ─────┐   ┌─ サービス層（非公開）──────────┐
│ Marketing-OS         │   │ 顧問デリバラブル：                    │
│ 需要予測構造診断      │   │   業界別ベンチマーク K 値            │
│ = ソルバーの Web 版  │   │   Price Adjustment 実係数・個社予測  │
│                      │   │   外部データパイプライン              │
└──────────────┘   └──────────────────────────┘
```

診断ツールは「OSS ソルバーの Web フロント」。サービスは「OSS ＋ 非公開資産で回す実務」。両方がこの OSS を中核として参照する。

## 公開するもの（本リポジトリ）

- **計算方法**：NBD 確率質量関数、K 同定アルゴリズム、コンセプトシェア集計、ユニットシェア／売上式。
- **選定の考え方**：数理 vs AI のメソッド選定決定木、ハイブリッド運用ルール。
- **調査テンプレート**：BP-10 の設問と集計方法。
- **思想**：なぜ数理 × AI か。

これらは**コモディティ化してよい知識**だ。公知の数式・モデル構造であり、隠しても競争力にはならない。むしろ公開して「編集可能な素材」にすることに価値がある。

## 公開しないもの（顧問デリバラブル）

- **業界別ベンチマーク K 値**：顧問先データの蓄積で得た、業界・カテゴリごとの典型 K。
- **Price Adjustment の実係数**：価格弾力性の実測係数。
- **個社予測**：特定企業向けの需要予測・事業シミュレーション。
- **外部データパイプライン**：予測を回すためのデータ取込・整備の仕組み。

これらは**顧問先データで蓄積した資産＝競争力の源泉**だ。ソルバー（計算方法）は公開しても、そこに入れる係数とデータは公開しない。包丁の作り方は公開するが、仕入れた食材とレシピの配合は店の中にある——という線引き。

## なぜ明示するのか

「タダで全部くれるのか、それとも囲い込むのか」を曖昧にしない。**方法論は公開し、実務は支援する**。この境界をコードとドキュメントで示すことが、信頼の担保になると考える。

- **試したい** → [Marketing-OS 需要予測構造診断](https://marketing-os.jp)（Web でソルバーを動かす）
- **頼みたい** → [Start-X 需要予測・事業シミュレーション](https://start-x.work/service/)（非公開資産で回す実務）

## 混入ゼロの原則

本リポジトリには、非公開資産（ベンチマーク K 値・実係数・顧問先情報）を**一切含めない**。examples のパラメータはすべて説明用のダミー値であり、実データではない。
