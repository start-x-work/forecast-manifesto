# 04. BP-10 設問テンプレート＋集計方法

BP-10（Buying Preference / 10 点配分法）は、販売実績のない新商品の**コンセプト受容度**を測る調査手法。各回答者に 10 票（ポイント）を候補ブランド／コンセプト間に配分してもらう。単一選択（「どれが一番好き？」）より、**相対的な選好の強さ**を捉えられる。

## 設問テンプレート

> 以下の商品コンセプトについてうかがいます。
> あなたが**次にこのカテゴリで買うとしたら**、どれをどれくらい選びそうですか。
> 合計が **10 になるように**、点数を配分してください。
> （すべてを 1 つに集中させても、複数に分けても構いません）
>
> | コンセプト | 配分（合計 10） |
> |-----------|----------------|
> | A（対象の新商品） | ___ |
> | B（競合） | ___ |
> | C（競合） | ___ |
> | …        | ___ |
> | **合計** | **10** |

### 設計上の注意

- **合計 10 を明示**する。端数計算がしやすく、10% 刻みの直感に合う。
- 選択肢には**対象コンセプトと主要競合**を並べる。「その他」は入れても集計から外せる。
- コンセプト提示文は全ブランドで**情報粒度を揃える**（片方だけ詳しいと票が偏る）。
- 価格を提示するかは調査目的で決める。価格影響は別途 Price Adjustment 係数で扱う。

## 集計方法：コンセプトシェア

対象ブランドの**平均得票率**をコンセプトシェアとする。回答者ごとに「対象への配分 ÷ その回答者の総配分」を求め、回答者間で平均する。

```
conceptShare = mean_i ( votes[i][target] / Σ_j votes[i][j] )
```

行ごとに正規化することで、合計が 10 でない無効票や棄権にも頑健になる。総配分が 0 の回答者はシェア 0 として扱う。

```ts
import { conceptShare } from "@forecast-manifesto/solver";

// 回答者 × ブランド の配分行列（列 0 が対象の新商品）
const votes = [
  [4, 3, 3],
  [6, 2, 2],
  [3, 4, 3],
];
const cs = conceptShare(votes);      // 対象ブランドの平均得票率
// 別ブランドを対象にする場合は列インデックスを渡す
const csB = conceptShare(votes, 1);
```

## 売上予測への接続

コンセプトシェアは、ユニットシェアの一項として使う。

```
ユニットシェア = 認知率 × 配荷率 × コンセプトシェア × 価格調整係数
売上          = 市場規模 × ユニットシェア × 単価
```

- **認知率**：どれだけの人が商品を知っているか。
- **配荷率**：どれだけの店・チャネルで買えるか。
- **コンセプトシェア**：知っていて買える人のうち、どれだけが選ぶか（BP-10 由来）。
- **価格調整係数（Price Adjustment）**：基準価格で 1.0。値引き・割高の乗数。**業界別の実係数は非公開資産**（→ [05-boundaries.md](./05-boundaries.md)）。
