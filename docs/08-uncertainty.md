# 08. 不確実性の定量化 — 点ではなく、幅で語る

同じ観測から同定したパラメータでも、**母数が小さければ推定には幅がある**。その幅を隠して点だけを見せるのは、予測を「当てるゲーム」にする第一歩だ。v1.2 からは全推定に区間を付与できる。

## 方式：パラメトリック・ブートストラップ

推定したパラメータで**個票を再生成 → 再推定**を `iterations` 回繰り返し、得られた推定値のパーセンタイルを区間とする。

```
観測 → 推定 (M, K̂)
        ↓ NBD(M, K̂) から nCustomers 人を再生成
        ↓ 浸透率・M を再計測 → K を再同定       × iterations 回
        ↓
     K̂ の分布 → [5%点, 95%点] = 90% 区間
```

- 乱数は **mulberry32（シード固定・依存ゼロ）**。同一シードで区間は完全に再現される。
- 再生成はポアソン・ガンマ・ベルヌーイの合成サンプラ（外部依存なし）。
- **フルベイズ（MCMC）はスコープ外**。事後分布そのものが必要な場合は [PyMC-Marketing](https://github.com/pymc-labs/pymc-marketing) に委譲する。

## API（点推定 API は不変）

既存の点推定関数のシグネチャ・戻り値は一切変えていない。`*WithInterval` の**追加のみ**。

```ts
// solver
identifyKWithInterval(M, penetration, { nCustomers: 2000 })
// → { K, ci: [lo, hi], samples, skipped }

// clv
fitBgNbdWithInterval(rfm, { iterations: 200, seed: 1 })
// → { params, ci: { r, alpha, a, b }, samples }
clvWithInterval(c, p, gg, { horizonMonths: 12, monthlyDiscount: 0.01, margin: 0.3 })
// → { point, p5, p50, p95 }
summarizeWithInterval(rfm)
// → summarize の各値に 90% 区間
```

既定値：`iterations = 200`／`seed = 1`／`level = 0.9`（→ [5%, 95%]）。

## 反復数と区間の安定性

M=1.4・浸透率（K=0.75 相当）・n=2,000 で、シードを変えて区間幅がどれだけブレるかを実測した：

| iterations | 区間幅（シード 1 / 2 / 3） | シード間のブレ |
|---:|---|---|
| 50 | 0.133 / 0.179 / 0.178 | 大（±15%） |
| 100 | 0.139 / 0.177 / 0.157 | 中（±12%） |
| **200（既定）** | **0.147 / 0.154 / 0.156** | **小（±3%）** |
| 400 | 0.149 / 0.149 / 0.152 | 極小（±1%） |

**200 反復あれば区間はほぼ安定する**。速報値は 50 で出し、レポートに載せる数字は 200 以上で確定させる、という使い分けを推奨する。

## 区間の読み方

1. **幅が広い＝データが足りない**。n=500 と n=10,000 では K の 90% 区間幅が約 4 倍違う（`npm run example:interval` で確認できる）。幅が広いうちは点で意思決定しない。
2. **CLV の分布は右に裾を引く**。中央値 < 点推定（期待値）になるのが普通。予算会話は p5〜p95 で行う。
3. **区間はモデル内の不確実性**しか表さない。モデル構造自体の誤り（例：非契約型でないのに BG/NBD を使う）は区間に現れない——それは [検証（validate）](./05b-clv.md) と使い分けの問題。

## 計算量の目安

`iterations × 顧客数 > 1e7` で警告（`warning` フィールド）が返る。CDNOW 全量（2,357 顧客）× 200 反復の `fitBgNbdWithInterval` は数十秒〜1分程度。急ぐときは顧客をサンプリングするか iterations を下げる。

## 注記：MAPE のゼロ除外

検証モジュール（validate）の `mape` は `actual = 0` のペアを**除外して**計算する（ゼロ割回避）。ゼロが多いデータでは MAE / RMSE を併記すること。
