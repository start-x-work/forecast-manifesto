# 06. 増分検証（インクリメンタリティ）— 「効いた」をどう確かめるか

広告・施策の効果は、**やらなかった世界との差分**でしか定義できない。相関（出稿が増えた月に売上も増えた）は証拠にならない——出稿は需要期に増えるからだ。本ページは増分検証の**設計の考え方だけ**を扱う。コードは書かない：実行は [GeoLift](https://github.com/facebookincubator/GeoLift)（合成対照群）や [Meridian](https://github.com/google/meridian)（MMM）等の実績あるOSSに委ね、**実行はクライアント名義**を原則とする（データ・アカウント・意思決定の主体はクライアントに残す）。

## 決定木：どの検証方式を選ぶか

```
Q1. ユーザー単位でランダム分割できるか？
 │   （自社アプリ・会員制・広告プラットフォームのRCT機能が使える）
 ├─ できる → プラットフォーム内テスト
 │            （Conversion Lift / Brand Lift / 自社ABテスト）
 │            最も感度が高い。まずこれを検討する
 └─ できない（店頭売上・指名検索・オフライン流入が主目的）
     Q2. 地域で分割できるか？（地域別の出稿制御と売上計測が可能）
      ├─ できる → GeoLift（合成対照群による地域テスト）
      │            テスト地域に出稿を増やし/止めて、
      │            残り地域の加重合成で「やらなかった世界」を作る
      └─ できない → 時系列断・MMM（Meridian等）で近似
                     ※因果の強度は落ちる。「参考値」と明示して使う
```

## 最小設計チェックリスト

テストを始める前に、以下が埋まらないなら**まだ始めない**：

- [ ] **期間**：効果の立ち上がり＋計測に十分な長さ（目安：4〜8週。購買サイクルが長い商材はそれ以上）
- [ ] **出稿量の差**：テスト群と対照群の出稿差が十分大きいか（検出したい効果より小さい差は検出できない）
- [ ] **地域数**：テスト地域は最低でも複数。1地域だけでは地域固有ショックと区別できない
- [ ] **事前トレンドの並行性**：テスト開始前の期間で、テスト群と対照群（または合成対照）の売上が並行して動いているか。並行していないなら対照の作り直し
- [ ] **汚染の遮断**：テスト地域への配信が対照地域に漏れないか（配信設定・商圏の重なり）
- [ ] **同時イベントの棚卸し**：期間中の値引き・PR・競合の動きを記録しておく

## 結果の読み方

- **iROAS（増分ROAS）**＝増分売上 ÷ 追加出稿費。全体ROASではなく**増分**で語る。プラットフォーム管理画面のROASは増分ではない。
- **信頼区間で判断する**。点推定のiROASが1.2でも、区間が [0.6, 1.8] なら「1を下回る可能性が相応にある」と読む。[docs/08](./08-uncertainty.md) の「幅で語る」と同じ規律。
- **「有意でない」＝「効果ゼロ」ではない**。検出力不足（期間・出稿差・地域数が足りない）の可能性をまず疑う。逆に、有意でないのに「方向は正しい」と言い張って継続するのは、当てるゲームへの逆戻り。
- **構造への接続**：増分が確認できたら、その効果を A1/A2 の構造（浸透率が動いたのか、頻度が動いたのか）に分解して記録する。次の予測の入力になる。
