# 02. メソッド選定の決定木

「数理モデルか、AI 時系列か」は、対象データと予測用途で決まる。以下の決定木に従う。

```
Q1. 予測対象に販売実績データはあるか？
 │
 ├─ ない（新商品・新規事業・新立地）
 │   → 数理モデル（NBD ＋ BP-10）
 │     コールドスタート耐性・説明可能性が投資合意に必要
 │
 └─ ある
     │
     Q2. 予測の用途は？
      │
      ├─ 戦略判断（投資・年間計画・価格改定）
      │   → 数理モデルで構造診断（M・K 同定）→ 垂直/水平の配分決定
      │
      └─ 運用（日次発注・在庫・生産）
          │
          Q3. データ品質ゲートを通るか？（週次連続性・欠損率）
           │
           ├─ 通る   → AI 時系列（外部データ取込・自動追従）
           │
           └─ 通らない → まず数理で近似運用 ＋ データ整備
```

## 各分岐の意味

### Q1：実績データの有無

販売実績がなければ機械学習は学習対象を持たない。NBD ＋ BP-10 なら、コンセプト調査（BP-10）と市場の構造仮定から予測を組める。→ [03-nbd-model.md](./03-nbd-model.md) / [04-bp10.md](./04-bp10.md)

### Q2：用途（戦略 vs 運用）

実績があっても、**戦略判断には数理の構造診断を優先**する。M・K を同定すれば、市場の天井（浸透率の上限 `1 - e^(-M)`）と、需要を垂直（既存顧客の頻度増）／水平（新規顧客の獲得）どちらに伸ばせるかが読める。運用の自動化とは別レイヤーの意思決定だ。

### Q3：データ品質ゲート

運用を AI に委ねる前提は品質。**週次の連続性**（欠測週がないか）と**欠損率**をゲートにする。通らなければ、まず数理で近似運用しながらデータ基盤を整える。品質の悪いデータへの機械学習は、それらしく外れる。

## ハイブリッド運用ルール

```
四半期ごとに K 値を再同定する。
K の変化が閾値を超過 → 市場構造が変化したシグナル
                    → AI モデルの再学習・特徴量見直しのトリガー
```

数理の K を、AI モデルの健全性を測る監視ダイヤルとして常時回す。

## 物理制約の連立（必須）

予測は必ず**物理制約と連立**させる。

- **スループット上限**：生産・供給キャパシティを超える予測は、超えた分を割り引く。
- **アクセス摩擦係数**：立地・配荷の届きにくさによる割引をシミュレートする。

予測がキャパシティを突き抜けるエラーを防ぐ。売上予測は「需要」と「供給できる量」の小さい方で頭打ちになる——この当たり前を数式に落とす。

実装例：[`examples/new-product-forecast.ts`](../examples/new-product-forecast.ts) の供給上限チェックを参照。
